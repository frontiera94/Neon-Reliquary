System Architecture & UX Specification: "Neon Reliquary" Pathfinder 1e Companion

1. Project Goal

Develop a high-performance, responsive web application (React/TS) that serves as a digital companion for Pathfinder 1st Edition. The app features a "Neon Noir" Dark Fantasy aesthetic, specialized for both PC and mobile use, with automated mechanics and a 2D interactive dice system.

2. Visual Identity (Neon Reliquary Style)

Based on the provided reference images, the UI uses a high-tech, dark terminal dashboard aesthetic.

Background: Deep Onyx/Carbon (#0a0a0f) with a clean, flat, matte finish.

Primary Color: Neon/Electric Cyan (#00d1ff) for primary buttons, active states, filled resource squares, and highlighted numeric values.

Secondary Accents:

Headers/Labels: Amber/Gold (#ffb800) for section titles.

Danger/Damage: Neon Red (#ff2b2b) for HP depletion and critical warnings.

Typography: Clean Sans-Serif (e.g., "Inter", "Rajdhani") for UI; Monospace for numeric values.

3. Tech Stack & Engineering

To ensure a robust and scalable architecture, the following stack is selected:

Language: TypeScript (TS). Mandatory for strict typing of complex Pathfinder mechanics (e.g., dynamic modifiers, spell slots, and condition stacking).

Frontend Framework: React 18+ with Vite for fast bundling and HMR (Hot Module Replacement).

State Management: Zustand. Chosen for its lightweight footprint and excellent performance in handling frequent state updates (like HP changes or dice rolls) without unnecessary re-renders.

Styling: Tailwind CSS. Enables rapid development of the "Neon Noir" aesthetic using utility classes and easy management of responsive breakpoints (Desktop vs Mobile).

Animations: Framer Motion. Used for smooth UI transitions between tabs and the 2D dice "shake/rotate" logic.

Data Persistence: IndexedDB (via Dexie.js) or localStorage for session persistence.

Extraction Agent (Backend/Local): Node.js or Python. To be used for the PDF-to-JSON parsing pipeline using libraries like pdf-parse or PyMuPDF.

4. Data & State Architecture

Source: Standardized JSON files generated by a dedicated Extraction Agent.

Multi-Character: Global state management (Zustand) for instant character switching.

Persistence: Full session state (HP, spent slots, toggled buffs, prepared spells, active conditions) saved to localStorage.

6. Functional Modules

A. Character Selection Screen ("Choose Your Relic")

Layout: Full-screen horizontal carousel of character portraits with high-contrast Cyan borders.

Call to Action: A large Cyan button labeled "MANIFEST CHARACTER".

B. Status Tab

Vitality Core: Huge Cyan HP display with tactile [ - ] and [ + ] buttons.

Stats Grid: Multi-tile grid for AC, Saves, and Ability Score modifiers.

Daily Resources: Horizontal rows for class resources (Ki, Rage, Arcane Pool) using tactile Cyan squares.

C. Spells Tab (The Repository)

Spell Library: List of all known spells grouped by level (0-9).

Preparation Mechanic: Toggle to "Prepare" specific spells for the day.

Arcane Failure Tracker: Displays the % chance of spell failure based on equipped armor.

D. Combat Tab (Engagement Mode)

Active Protocol Bar: Toggles for global buffs (Power Attack, Deadly Aim, Rage).

Weapon Cards: Primary attack/damage tiles with automated calculation of all active toggles.

Combat Spells: A section listing only Prepared Spells for quick casting.

E. Companion Tab

Secondary Sheet: Simplified Status and Combat tabs for Animal Companions, Familiars, or Eidolons.

7. 2D Interactive Dice System (Overlay)

Trigger: Any numeric value featuring a small dice icon.

Dice Tray: Bottom-docked widget to add "On-the-Fly" dice or manual flat modifiers.

Animation: 2D "spin and shake" icons with a Neon Cyan glow on the result.

8. Special Game Logic

Critical Threat: Automatic "Confirm Critical" button on threat.

Dynamic Recalculation: Instant updates to Attack/Damage/AC based on active Toggles and Conditions.

9. Automated Data Pipeline (The "Extraction Agent")

To ensure data integrity and zero-manual entry for players, a specific Backend/Local Agent must be instantiated.

Input: PDF export from PCGen (Standard sheet).

Processing Logic:

OCR/Text Extraction: Parse structured text from PCGen PDF layout.

Normalization: Map raw text (e.g., "STR 18 (+4)") into structured JSON fields (stats: { str: { value: 18, mod: 4 } }).

Conflict Resolution: Identify and flag unknown abilities or ambiguous text for Admin validation.

Output: A standardized .json file formatted for the "Neon Reliquary" schema.

Role: This agent acts as the gateway; it is controlled solely by the Administrator (Admin-Only Validation Flow).
