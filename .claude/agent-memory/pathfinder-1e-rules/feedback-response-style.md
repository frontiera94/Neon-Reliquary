---
name: Response Style Preference
description: User wants direct answer first, then rule+source, then worked example, then caveats; no padding
type: feedback
---

Structure every rules answer as:
1. Direct answer (confirm or deny the premise, one sentence)
2. Rule text or close paraphrase + source (book abbreviation + page or class feature name)
3. Worked example with actual character stats when relevant
4. Caveats: RAW vs. RAI notes, GM discretion flags, data issues found in JSON

**Why:** User is a rules-verification workflow, not a learning context. They want signal, not explanation of basics they already know.

**How to apply:** Lead with the verdict. Do not bury the answer in setup. Tables are welcome for multi-row comparisons (e.g., Judgment bonus tiers, attack sequence matrices).
